## 三、负样本质量问题（最严重）

### 3.1 负样本来源构成

| 来源 | 数量 | 占比 | 可靠性 |
|------|------|------|--------|
| 明确阴性文献（PMID 36162894等） | 58 | **2.3%** | ⭐⭐⭐⭐⭐ |
| 骨有害药物（糖皮质激素→骨坏死） | 191 | 7.7% | ⭐⭐⭐⭐ |
| 跨类别排斥 | 92 | 3.7% | ⭐⭐⭐ |
| CTD药物×非TMJOA疾病 | 590 | 23.7% | ⭐⭐ |
| 随机噪声基线 | **1561** | **62.6%** | ⭐ |

**🔴 核心问题：62.6%的负样本是随机噪声**

这意味着：
- 模型可能在学"随机药物-疾病组合不相关"而非"特定药物对TMJOA无效"
- 随机噪声过多→模型区分度下降（正样本和负样本都太宽泛）
- 真正的"硬负样本"（明确对TMJOA无效的药物）仅占2.3%

### 3.2 假阴性风险

| 药物 | 是否被标记为负样本 | 风险 |
|------|-------------------|------|
| Metformin | ✅ 是（合成负样本） | 🔴 **高风险** — 大量OA机制研究表明Metformin对软骨保护有效 |
| Resveratrol | ✅ 是 | 🔴 **高风险** — 已被证实促进软骨细胞存活 |
| Curcumin | ✅ 是 | 🔴 **高风险** — 多篇OA抗炎RCT |
| Rapamycin | ✅ 是 | 🔴 **高风险** — mTOR抑制剂促进自噬，软骨保护 |
| Quercetin | ✅ 是 | 🟡 中等风险 |
| Omega-3 | ✅ 是 | 🟡 中等风险 |
| Vitamin D | ✅ 是 | 🟡 中等风险 — 但骨质疏松是正样本 |
| Statin | ✅ 是 | 🟡 中等风险 |

**这些"重定位明星"被标记为负样本是错误的。** 它们在OA模型中有明确保护证据，对TMJOA也应是潜在正样本（或至少不应是负样本）。

## 四、外部数据库导入问题

### 4.1 CTD导入（513样本）

**问题1: CTD中TMD仅3个关联**
- CTD主要覆盖RA/OA/Osteoporosis，TMJOA研究空白
- 导致TMJOA特异性信号被OA通用信号淹没

**问题2: CTD的"marker/mechanism"证据≠治疗有效性**
- CTD记录的是基因-药物-疾病关联（如"某药物影响某基因，该基因与某疾病相关"）
- 不等于该药物对该疾病有治疗作用
- 将mechanism证据赋label=0.3可能过度乐观

**问题3: CTD的"inference"关联可信度低**
- InferenceScore基于文献共现算法
- 大量推断关联可能是假阳性
- 建议：仅使用DirectEvidence=therapeutic的CTD数据

### 4.2 ChEMBL导入（33样本）

- 数量少，影响有限
- 但来源可靠（FDA临床阶段数据）
- 保留

### 4.3 医学知识库（83样本）

- 基于临床指南共识
- 较为可靠
- 保留

## 五、数据泄露风险

### 5.1 同一PMID多药物-疾病对

多个样本可能来自同一篇文献（同一PMID），这导致：
- Train/Val/Test划分时的信息泄露
- 同一篇RCT的多个药物-疾病对可能被分到不同split
- **建议**: 按PMID分层划分，而非按drug分层

### 5.2 CTD多关联膨胀

单个CTD药物可能有数百个疾病关联，导致：
- 某些药物在训练集中出现数百次
- 模型可能过度学习这些"高频药物"的特征
- **建议**: 每个药物-疾病对只保留最高置信度的一个关联

### 方案C: 当前版本（v3.9c）风险评估

直接使用v3.9c（3641样本）的风险：
- **高风险**: 假阴性（重定位明星被错误标记为负）
- **中风险**: 模型过拟合OA通用模式
- **中风险**: 随机噪声降低区分度
- **低风险**: 总体规模充足

**建议**: 如果必须使用v3.9c，需：
1. 手动修正重定位明星的label（从0.0改为0.3-0.5 soft positive）
2. 在Test集中保留纯TMJ核心样本作为held-out评估
3. 使用类别权重（正样本weight=1.5×, 负样本weight=1.0×）
4. 报告结果时区分"TMJOA特异性预测"和"OA通用预测"

## 七、对GNN训练的具体建议

### 7.1 网络构建

建议三层异构图：
```
药物层 (Drug) → 靶点层 (Target/Protein) → 疾病层 (Disease)
                ↓
           机制层 (Pathway/GO)
```

### 7.2 损失函数

```python
# 加权BCE，对TMJ核心正样本额外加权
weights = torch.where(label > 0.5, 2.0, 1.0)  # 强阳性加倍权重
weights = torch.where(tmj_relevance == 'direct', weights * 1.5, weights)

loss = F.binary_cross_entropy_with_logits(
    pred, label, weight=weights
)
```

### 7.3 评估策略

**必须分层评估**：
1. **Held-out TMJ核心测试集**: 只包含TMJOA/TMJ/TMD直接证据的样本
2. **跨病类比测试集**: OA/Knee OA等类比样本
3. **重定位明星评估**: 单独测试Metformin/Rapamycin/Resveratrol等的预测分数
